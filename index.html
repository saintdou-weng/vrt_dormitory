▶ 一、系統概述 System Overview
本系統用於 VRT 宿舍的搬入/搬出線上申請與審批管理，並整合 Telegram Bot 通知。
審批流程：申請人填表提交 → Telegram 通知群組 → Paul 點按鈕核可或退件 → 申請人收到私訊

功能分頁：
📊 總覽 — Dashboard 統計（待審核/已核可/已退件/在住房間）
📋 申請表單 — 搬入/搬出申請、物品勾選、儲存/匯出/列印/提交
👥 在住總表 — 即時在住人員名單（已核可搬入 − 已核可搬出）
🗂️ 記錄查詢 — 歷史記錄篩選、Excel 匯出、Excel 匯入、列印
⚠️ 建議使用 Chrome 瀏覽器，且保持網路連線以確保 GAS 同步正常。
▶ 二、搬入操作 Move-in Workflow
1
切到「📋 申請表單」，上方選 「🏠 搬入」
2
填寫必填欄位：姓名、性別、工號、部門、房號、搬入日期、原因
3
宿舍物品：預設 4 項，勾選「領取 ✓」並選狀態。可新增/刪除項目
4
填寫 簽核欄位：申請人、審核人、核可人姓名
5
點 「提交搬入」 — 系統自動檢查是否有重複待審核申請後送出
💡 系統會檢查同一工號是否已有「待審核」申請，重複時會阻止提交。
▶ 三、搬出操作 Move-out Workflow
1
上方選 「🚪 搬出」，輸入工號後按 Enter 自動帶出在住資料
2
逐一勾選物品「歸還」，選擇狀態，損壞/遺失請填備註
3
提交時系統自動檢查物品完整性，有問題會彈出確認視窗
▶ 四、Excel 匯入 Import Excel
1
在「🗂️ 記錄查詢」頁，點 「📤 匯入 Excel」 選擇檔案
2
系統自動讀取 Excel 第一個工作表，解析欄位後逐筆提交至 GAS
3
匯入格式：第一列為標題，欄位依序為 類型、姓名、工號、性別、部門、職位、電話、房號、日期、原因
⚠️ 匯入資料會直接寫入 Google Sheet（狀態為「待審核」），請確認資料正確。
▶ 五、後端部署（IT 管理員）
①
建立 Google Sheet → 不需手動建表（GAS 第一次執行時自動建立 Dorm_Records）
②
Apps Script 中貼上 Code.gs 全部程式碼
③
部署為 Web App（Execute as: Me, Access: Anyone）
④
將部署 URL 貼到 dorm.html 第一行 GAS_URL 變數
⑤
在 GAS 編輯器執行 registerWebhook() 函式一次，完成 Telegram Webhook 設定
⑥
推送 dorm.html 到 GitHub Pages → saintdou-weng/vrt_dormitory
